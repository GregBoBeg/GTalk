Behind the Scenes

The Application

GTalk is a chat application developed in vanilla Javascript, which uses Node.js, Express and Socket.IO for backend communication between users.

Front-end

The Welcome Page allows users to choose which chat discussion they wish to join.

Once the discussion topic is selected, the user can chat in real-time with other users about that topic.

Back-end

The server-side code emits signals as users join/leave discussions. When a signal is detected, the user is added to (or removed from) the socket. A message is then broadcasted back out over the socket, notifying all other users that someone has joined/left the discussion.

When a user posts a message, the server-side code detects the message and broadcasts it back out over the socket to all other users in that discussion, thus facilitating real-time chat.

The Deployment

The GTalk app is deployed on the Heroku cloud platform. The project is maintained with version control, using GIT, and pushed to a Github repository. Heroku is configured with a pipeline to automatically recognize and deploy changes from the Github repository.

Test-Driving GTalk

To test drive the app, open multiple copies of the app's Welcome Page (index.html). This can be done on multiple devices or on a single device with multple browser windows.

On each Welcome Page, enter a unique Username. Also, be sure to select the same discussion topic on each Welcome Page. This will allow real-time chatting between each browser window.
